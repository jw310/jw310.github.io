JavaScript 實用技巧

介紹一些寫 JavaScript 時注意的地方

宣告變數記得給預設型別

觀看程式碼時，可以快速了解變數型別。

Code

寫判斷時使用 === 跟 !==

避免比較時自動轉換型別，出現 "1" == 1 為 true 的情況。

Code

使用 return 代替 if else

適時用 return 代替 if else 可以讓程式可讀性更高。

Code

盡量不要直接修改 DOM 的 CSS

要改變 DOM 的 style 時，盡量避免直接用 JavaScript 操作 CSS! ，可使用 add 或 remove 的 class 方法去增加及刪除樣式。

Code

簡單動畫少用函式庫

簡單的互動，可以交由 css3 的 transition 來處理，在行動裝置上會順暢很多。

Code

篩選資料時用 filter()

多利用 filter 來篩選資料，提高 code 的可讀性。
ps. filter()：回傳符合條件的元素，得到一個新陣列。

Code

多利用 console.log 來 debug

開發階段時可多用 console.log 來確認變數的內容及程式的運行流程(尤其非同步執行時)，不過記得程式要上線時，把測試的 console.log 給刪掉，減少效能消耗。

Code

整合 <script> 載入的進入點

主程式不要分檔寫，以確保程式流程。

Code

不要操作原始陣列

因為陣列有傳參考的特性，所以對傳入的 array 直接操作會修改到原始資料。因此當傳入 array 進去函式裡面進行操作的時候，我們應該先拷貝一份陣列！為了避免這種情況我們應該要用 slice(0) 先複製一份陣列再來操作。

Code

不要 bind this

this 的指向一直都是 JavaScript 中最令人頭疼的問題，尤其是為了用 function 來模擬 class 的使用上面，常常需要讓綁定事件內 this 指向到 function(class)的實體上，所以常會使用到 bind，使用 bind 會造成函式的記憶體增加，我們應該使用變數來儲存 this 來調用，而不是 bind。

Code
